
# REST API и JSON

## 1. Методы REST API

| Метод | Назначение |
|-------|------------|
| GET | Получение данных с сервера (чтение ресурса) |
| POST | Создание нового ресурса на сервере |
| PUT | Полное обновление существующего ресурса |
| PATCH | Частичное обновление существующего ресурса |
| DELETE | Удаление ресурса с сервера |
| HEAD | Получение только заголовков ответа без тела |
| OPTIONS | Получение информации о доступных методах для ресурса |

## 2. Отличия методов

| Сравнение | Описание |
|-----------|----------|
| **PUT vs PATCH** | PUT заменяет весь ресурс целиком, PATCH обновляет только указанные поля |
| **GET vs HEAD** | GET возвращает данные + заголовки, HEAD возвращает только заголовки (без тела ответа) |

## 3. Статус-коды HTTP

| Код | Описание |
|-----|----------|
| 1XX | Информационные коды (запрос получен, обработка продолжается) |
| 2XX | Успешные коды (запрос успешно обработан) |
| 3XX | Коды перенаправления (требуется дополнительное действие) |
| 4XX | Коды ошибок клиента (ошибка в запросе) |
| 5XX | Коды ошибок сервера (ошибка на стороне сервера) |

## 4. JSON

| Вопрос | Ответ |
|--------|-------|
| **Как расшифровывается JSON?** | JavaScript Object Notation (нотация объектов JavaScript) |

## 5. Преимущества JSON

| Формат | Преимущества JSON |
|--------|-------------------|
| **JSON vs XML** | Более компактный, быстрее парсится, проще читается, меньше избыточности |
| **JSON vs YAML** | Более строгий синтаксис, лучше для API, меньше ошибок парсинга, быстрее обработка |

## 6. JSON Schema

| Понятие | Описание |
|---------|----------|
| **JSON Schema** | Спецификация для описания структуры и валидации JSON-документов |

## 7. Сериализация и десериализация JSON

| Процесс | Описание |
|---------|----------|
| **Сериализация** | Преобразование объекта/структуры данных в JSON-строку для передачи или хранения |
| **Десериализация** | Преобразование JSON-строки обратно в объект/структуру данных для использования в программе |
