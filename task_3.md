
# Базы данных: реляционные vs нереляционные

## 1. Отличия реляционных БД от нереляционных

| Характеристика | Реляционные БД | Нереляционные БД (NoSQL) |
|----------------|----------------|---------------------------|
| **Структура данных** | Таблицы с фиксированными схемами | Документы, ключ-значение, графы, колонки |
| **Связи между данными** | Строгие связи через внешние ключи | Гибкие связи или их отсутствие |
| **ACID-транзакции** | Полная поддержка | Ограниченная поддержка |
| **Масштабируемость** | Вертикальная (увеличение мощности сервера) | Горизонтальная (добавление серверов) |
| **Схема** | Фиксированная схема | Динамическая схема |
| **Язык запросов** | SQL | Специфичные языки запросов |
| **Примеры** | MySQL, PostgreSQL, Oracle | MongoDB, Redis, Cassandra |

## 2. Плюсы и минусы

### Реляционные БД

| Плюсы | Минусы |
|-------|--------|
| Гарантия целостности данных | Сложность масштабирования |
| ACID-транзакции | Фиксированная схема |
| Стандартизированный SQL | Высокие требования к ресурсам |
| Зрелость и надежность | Сложность при изменении схемы |
| Поддержка сложных запросов | Ограничения производительности |

### Нереляционные БД (NoSQL)

| Плюсы | Минусы |
|-------|--------|
| Горизонтальное масштабирование | Ограниченная поддержка ACID |
| Гибкая схема данных | Сложность сложных запросов |
| Высокая производительность | Меньше инструментов |
| Простота разработки | Потенциальные проблемы с консистентностью |
| Подходит для больших данных | Необходимость изучения новых технологий |

## 3. Отличия СУБД между собой

### MySQL vs PostgreSQL

| Характеристика | MySQL | PostgreSQL |
|----------------|-------|------------|
| **Тип лицензии** | Двойная (GPL + коммерческая) | Open Source (PostgreSQL License) |
| **Производительность** | Быстрее для простых запросов | Лучше для сложных аналитических запросов |
| **Поддержка стандартов** | Частичная поддержка SQL стандартов | Полная поддержка SQL стандартов |
| **Типы данных** | Ограниченный набор | Богатый набор (JSON, массивы, пользовательские типы) |
| **Индексы** | B-tree, Hash, Full-text | B-tree, Hash, GIN, GiST, SP-GiST, BRIN |
| **Репликация** | Master-Slave, Master-Master | Streaming replication, Logical replication |
| **JSON поддержка** | С версии 5.7 | Нативная поддержка с версии 9.2 |
| **Полнотекстовый поиск** | MyISAM, InnoDB | Tsearch2, полнотекстовые индексы |
| **Расширяемость** | Ограниченная | Высокая (пользовательские функции, типы) |

### Другие популярные СУБД

| СУБД | Особенности | Лучше всего подходит для |
|------|-------------|-------------------------|
| **Oracle** | Коммерческая, высокая производительность, богатый функционал | Крупные корпоративные системы |
| **SQL Server** | Microsoft экосистема, интеграция с .NET | Windows-приложения, Microsoft экосистема |
| **SQLite** | Встраиваемая, файловая БД | Мобильные приложения, прототипирование |
| **MongoDB** | Документная NoSQL БД | Веб-приложения, контент-системы |
| **Redis** | In-memory, ключ-значение | Кэширование, сессии, очереди |
| **Cassandra** | Колоночная NoSQL БД | Большие данные, высокая доступность |
